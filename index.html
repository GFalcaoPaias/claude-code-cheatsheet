⬇ ADVANCED — Power users & automation
🎣 Hooks — Automate Everything
Event Lifecycle (in order)
SessionStartSession begins (matcher: "resume")
InstructionsLoadedCLAUDE.md loaded
UserPromptSubmitBefore prompt processed ✦ block
PreToolUseBefore any tool runs ✦ block
PermissionRequestBefore permission prompt ✦ block
PostToolUseAfter tool succeeds
PostToolUseFailureAfter tool fails
NotificationSystem notification (matcher: "permission_prompt")
SubagentStart/StopAgent lifecycle (matcher: "Explore")
StopClaude stops responding ✦ block
TeammateIdleAgent waiting (exit 2 to block)
TaskCompletedTask done (exit 2 to continue)
ConfigChangeSettings changed ✦ block
WorktreeCreateNew worktree (non-0 exit blocks)
PreCompact/PostCompactContext compaction
SessionEndSession closes
Hook Types
commandShell script — stdin JSON, stdout JSON, exit 0/2
httpPOST to URL — 2xx=ok, other=warn
promptLLM evaluates — returns {ok, reason}
agentSubagent with Read/Grep/Glob access
Block Mechanisms
exit 2Block from command hook
permissionDecision: "deny"Block PreToolUse
decision: "block"Block Stop/UserPromptSubmit
action: "decline"Block Elicitation
Config Structure
{ "hooks": { "PreToolUse": [{ "matcher": "Bash(git push*)", "hooks": [{ "type": "command", "command": "./check-branch.sh", "timeout": 30 }] }], "Stop": [{ "hooks": [{ "type": "command", "command": "notify-send 'Claude done'" }] }] } }
Matcher Patterns
"Bash"All Bash tool calls
"Edit|Write"Edit or Write
"mcp__.*"Any MCP tool (regex)
"Bash(git push*)"Specific command pattern
"resume"SessionStart on resume
"Explore"Explore subagent start
"auto"PreCompact auto-triggered
Hook Locations
~/.claude/settings.jsonAll projects (personal)
.claude/settings.jsonProject (shared)
.claude/settings.local.jsonProject (local)
.claude/rules/Path-based rules

Practical Examples
# Notify when Claude finishes (Stop event) "Stop": [{"hooks": [{"type": "command", "command": "notify-send 'Claude done'"}]}] # Auto-approve safe reads (PermissionRequest) "PermissionRequest": [{"hooks": [{"type": "command", "command": "./auto-approve.sh"}]}] # Block pushes to main (PreToolUse) "PreToolUse": [{"matcher": "Bash(git push*)", "hooks": [{"type": "command", "command": "./check-branch.sh"}]}] # LLM safety check (prompt hook) "PreToolUse": [{"matcher": "Bash", "hooks": [{"type": "prompt", "prompt": "Is this safe? $ARGUMENTS", "model": "claude-haiku-4"}]}]
🔐 Permission Rules
Permission Modes
default prompts
acceptEdits auto-accept edits
plan read-only
dontAsk deny unless allowed
bypassPermissions skip all
Rule Syntax
Tool or Tool(specifier) allow / ask / deny arrays in settings.json
Bash Wildcards
Bash(*)All bash commands
Bash(npm run *)Prefix match
Bash(* install)Suffix match
Bash(git * main)Wildcard in middle
Bash(ls *)Word boundary (≠ lsof)
File Rules (gitignore syntax)
Read(.env)Specific file in cwd
Read(src/**)Relative path
Read(~/docs/*.pdf)From home dir
Edit(/src/**/*.ts)TypeScript files
Other Rules
WebFetch(domain:github.com)Domain allow
mcp__githubAll tools from server
mcp__github__list_prsSpecific MCP tool
Agent(Explore)Deny Explore agent
Example settings.json
"permissions": { "allow": ["Bash(npm *)", "Read(src/**)"], "ask": ["Edit"], "deny": ["Bash(git push *)", "Bash(rm *)"] }
